# 30-Day Traction-First Plan

## Positioning tested in parallel
- Funnel A: 1-click install ComfyUI workflows + dependencies.
- Funnel B: shareable ComfyUI workflow hub.

Both funnels use the same waitlist form and tracking model.

## Audience
- Broad ComfyUI users.
- Creators + beginners + power users.

## Goal
- Reach 100 active Discord testers in 30 days.

## Weekly execution cadence

### Week 1
- Launch both funnel pages.
- Start tracking events and dashboard review.
- Publish first content wave on X, Discord, Reddit.

### Week 2
- Optimize copy based on funnel conversion.
- Add first user quotes and pain-language proof.
- Run onboarding threads in Discord.

### Week 3
- Repeat distribution with refined angle.
- Focus on highest-converting message.
- Capture qualitative pain clusters.

### Week 4
- Review metrics and decide go/no-go.
- Freeze next build scope based on evidence.

## Messaging frameworks in practice

### Evan Spiegel lens
- One primary action per page.
- Minimal cognitive load.

### Russell Brunson lens
- Explicit funnel stages:
  - page view -> CTA click -> signup -> Discord join -> active

### Alex Hormozi lens
- Outcome language over features.
- Pain removed: setup time and dependency confusion.

### Elon Musk 5-step lens
1. Challenge requirements.
2. Delete unnecessary work.
3. Simplify.
4. Speed up.
5. Automate later.

### Gary Vee lens
- Daily short distribution from real user pain.
- Native-platform posting style.

### Y Combinator lens
- Talk to users directly.
- Do manual onboarding early for better signal.
